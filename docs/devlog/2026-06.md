### 문제 / 해결

- 문제:
  - React Query 도입으로 기존 로컬 `loading/error` 상태 처리와 API 호출 흐름을 일관되게 정리할 필요가 있었다.
  - 게시글 상세 조회 API 함수명이 문서의 `getPost(postId)` 기준과 구현의 `getPostById`로 불일치했다.
  - 게시글 삭제 성공 후 목록 query만 invalidate하면 삭제된 게시글 상세 query 캐시가 남을 수 있었다.
  - Issue 원문과 기존 문서 사이에 인증 API 경로 차이가 있어 최종 기준 정리가 필요했다.

- 해결:
  - 조회 API는 `useQuery`, 작성/수정/삭제/인증 요청은 `useMutation`으로 정리했다.
  - 게시글 상세 조회 API 함수명을 문서 기준인 `getPost`로 맞췄다.
  - 삭제 성공 시 게시글 목록 query invalidate와 함께 상세 query 캐시 제거를 반영했다.
  - 인증 API 경로는 `/api/auth/signup`, `/api/auth/login` 기준으로 문서와 구현을 정리했다.
